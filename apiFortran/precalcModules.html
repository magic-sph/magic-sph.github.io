Pre-calculations¶

preCalc.f90¶

Quick access

Routines:get_hit_times(), precalc(), precalctimes(), writeinfo()

Needed modules

  • horizontal_data (horizontal()): Module containing functions depending on longitude and latitude plus help arrays depending on degree and order
  • parallel_mod (rank()): This module contains the blocking information
  • useful (abortrun(), logwrite()): library with several useful subroutines
  • output_data: This module contains the parameters for output control
  • logic (l_time_hits(), l_to(), l_newmap(), l_storetpot(), l_heat(), l_mag_lf(), l_mag(), l_non_rot(), l_anelastic_liquid(), l_tomovie(), l_cond_ic(), l_storevpot(), l_anel(), l_non_adia(), l_storebpot(), l_probe(), l_lcr(), l_movie(), l_cmb_field(), l_chemical_conv(), l_precession(), l_storepot(), l_r_field(), l_diff_prec(), l_save_out(), l_temperature_diff(), l_dt_cmb_field()): Module containing the logicals that control the run
  • integration (rint_r()): Radial integration functions
  • precision_mod: This module controls the precision used in MagIC
  • init_fields (s_top(), omega_diff(), topxi(), n_xi_bounds(), tops(), s_bot(), botxi(), l_reset_t(), xi_bot(), n_s_bounds(), bots(), xi_top()): This module is used to construct the initial solution.
  • special (fac_loop(), l_curr()): This module contains all variables for the case of an imposed IC dipole, an imposed external magnetic field and a special boundary forcing to excite inertial modes
  • num_param: Module containing numerical and control parameters
  • physical_parameters (polind(), lffac(), ek(), epsc0(), tmagcon(), ktops(), epscxi0(), rho_ratio_ic(), osc(), pr(), sigma_ratio(), ktopxi(), epsc(), chemfac(), kbotxi(), ra(), n_r_lcr(), o_sr(), rho_ratio_ma(), nvardiff(), po(), diff_prec_angle(), buofac(), kbots(), opr(), radratio(), oek(), sc(), nvareps(), opm(), nvarvisc(), raxi(), prmag(), r_lcr(), epscxi(), corfac(), mode(), interior_model(), nvarcond(), ekscaled(), prec_angle(), rascaled()): Module containing the physical parameters
  • radial_functions (rho0(), dlvisc(), temp0(), ogrun(), rscheme_oc(), dlkappa(), transportproperties(), dllambda(), rgrav(), r_surface(), alpha0(), dltemp0(), beta(), divktemp0(), kappa(), visc(), r_cmb(), dbeta(), r(), radial(), r_icb(), dentropy0(), sigma(), lambda()): This module initiates all the radial functions (transport properties, density, temperature, cheb transforms, etc.)
  • constants: module containing constants and parameters used in the code.
  • truncation (n_r_ic_max(), n_cheb_ic_max(), minc(), n_phi_max(), l_max(), nalias(), n_r_max(), m_max(), lm_max(), n_cheb_max(), n_theta_max()): This module defines the grid points and the truncation

subroutine precalculations/precalc()¶

Purpose of this subroutine is to initialize the calc values, arrays, constants that are used all over the code. The stuff is stored in the common blocks. MPI: This is called by every processors.

Called from:magic
Call to:radial(), transportproperties(), horizontal(), rint_r(), abortrun(), logwrite()

subroutine precalculations/get_hit_times(t, n_t_max, n_t, l_t, t_start, t_stop, dt, n_tot, n_step, string_bn, time, tscale)¶
This subroutine checks whether any specific times t(*) are given on input. If so, it returns their number n_r and sets l_t to true. If not, t(*) may also be defined by giving a time step
Parameters:
  • dt [real,inout] :: Time step for output
  • t (n_t_max) [real,inout] :: Times for output
  • n_t [integer,out] :: No. of output times
  • l_t [logical,out] :: =.true. if output times are defined
  • t_start [real,inout] :: Starting time for output
  • t_stop [real,inout] :: Stop time for output
  • n_tot [integer,inout] :: No. of output (times) if no times defined
  • n_step [integer,inout] :: Ouput step in no. of time steps
  • string_bn [character,in]
  • time [real,in] :: Time of start file
  • tscale [real,in]
Options:

n_t_max [integer,in,optional/default=len(t)] :: Dimension of t(*)
Called from:

Call to:

abortrun()

radial.f90¶

Description

This module initiates all the radial functions (transport properties, density, temperature, cheb transforms, etc.)

Quick access

Variables:alpha0, alpha1, alpha2, beta, cheb_ic, cheb_int, cheb_int_ic, cheb_norm_ic, chebt_ic, chebt_ic_even, d2cheb_ic, d2temp0, dbeta, dcheb_ic, ddbeta, ddlalpha0, ddltemp0, ddlvisc, dentropy0, divktemp0, dlalpha0, dlkappa, dllambda, dltemp0, dlvisc, dr_fac_ic, epscprof, jvarcon, kappa, lambda, ndd_costf1, ndd_costf1_ic, ndd_costf2_ic, ndi_costf1_ic, ndi_costf2_ic, o_r_ic, o_r_ic2, ogrun, opressure0, or1, or2, or3, or4, orho1, orho2, otemp1, r, r_cmb, r_ic, r_icb, r_surface, rgrav, rho0, sigma, temp0, visc
Routines:finalize_radial_functions(), getbackground(), getentropygradient(), initialize_radial_functions(), polynomialbackground(), radial(), transportproperties()

Needed modules

  • chebyshev (type_cheb_odd())
  • parallel_mod (rank()): This module contains the blocking information
  • cosine_transform_odd
  • useful (abortrun(), logwrite()): library with several useful subroutines
  • algebra (prepare_mat(), solve_mat())
  • output_data (tag()): This module contains the parameters for output control
  • finite_differences (type_fd()): This module is used to calculate the radial grid when finite differences are requested
  • radial_der (get_dr()): Radial derivatives functions
  • radial_scheme (type_rscheme()): This is an abstract type that defines the radial scheme used in MagIC
  • mem_alloc (bytes_allocated()): This little module is used to estimate the global memory allocation used in MagIC
  • num_param (alph1(), alph2()): Module containing numerical and control parameters
  • cosine_transform_even
  • logic (l_newmap(), l_anelastic_liquid(), l_finite_diff(), l_tp_form(), l_mag(), l_heat(), l_single_matrix(), l_anel(), l_non_adia(), l_centrifuge(), l_temperature_diff(), l_cond_ic(), l_isothermal()): Module containing the logicals that control the run
  • chebyshev_polynoms_mod
  • physical_parameters: Module containing the physical parameters
  • constants (four(), three(), two(), sq4pi(), half(), one()): module containing constants and parameters used in the code.
  • truncation (n_r_ic_max(), fd_order(), n_r_max(), fd_stretch(), fd_order_bound(), n_cheb_max(), fd_ratio()): This module defines the grid points and the truncation

subroutine horizontal_data/gauleg(sinthmin, sinthmax, theta_ord, gauss, n_th_max)¶
Subroutine is based on a NR code. Calculates N zeros of legendre polynomial P(l=N) in the interval [sinThMin,sinThMax]. Zeros are returned in radiants theta_ord(i) The respective weights for Gauss-integration are given in gauss(i).
Parameters:
  • sinthmin [real,in] :: lower bound in radiants
  • sinthmax [real,in] :: upper bound in radiants
  • theta_ord (n_th_max) [real,out] :: zeros cos(theta)
  • gauss (n_th_max) [real,out] :: associated Gauss-Legendre weights
  • n_th_max [integer,in] :: desired maximum degree
Called from:

horizontal()
